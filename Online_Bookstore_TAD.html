1. Executive Summary

This Technical Architecture Document outlines the design and implementation of a cloud-native Online Bookstore application using microservices architecture deployed on Amazon EKS. The solution provides high availability, scalability, and security following AWS Well-Architected Framework principles.

5. Authentication & Security Architecture

5.1 Authentication Flow

  1. User Registration/Login → AWS Cognito User Pool
  2. Token Generation → JWT tokens issued by Cognito
  3. API Gateway → Token validation at ALB level
  4. Service Communication → Internal service mesh with mTLS
  5. Database Access → IAM database authentication

5.2 Security Best Practices Implementation

Security Layer Implementation AWS Service
Network Security VPC with private subnets, NACLs, Security Groups Amazon VPC
Web Application Firewall SQL injection, XSS protection, rate limiting AWS WAF
Identity & Access Multi-factor authentication, OAuth 2.0/OIDC AWS Cognito
Data Encryption Encryption at rest and in transit AWS KMS
Container Security Image scanning, runtime security Amazon ECR, AWS Security Hub
Secrets Management Database credentials, API keys AWS Secrets Manager

7. High Availability & Disaster Recovery

7.1 Production Environment HA Design

  • Multi-AZ Deployment: EKS worker nodes distributed across 3 availability zones
  • Database HA: RDS MySQL Multi-AZ with automatic failover
  • Load Balancing: Application Load Balancer with health checks
  • Auto Scaling: Horizontal Pod Autoscaler and Cluster Autoscaler
  • Backup Strategy: Automated RDS backups with 7-day retention

9. Deployment Strategy

9.1 CI/CD Pipeline

  1. Source Control: Git repository with feature branches
  2. Build: AWS CodeBuild for containerization
  3. Registry: Amazon ECR for container images
  4. Deployment: AWS CodePipeline with GitOps using ArgoCD
  5. Testing: Automated testing in QA environment
  6. Production: Blue-Green deployment strategy

10. Compliance & Governance

10.1 AWS Well-Architected Framework Alignment

  • Operational Excellence: Infrastructure as Code, Automated deployments
  • Security: Defense in depth, Encryption, IAM best practices
  • Reliability: Multi-AZ deployment, Auto-scaling, Health checks
  • Performance Efficiency: Right-sizing, Caching, CDN
  • Cost Optimization: Reserved instances, Auto-scaling, Resource tagging
